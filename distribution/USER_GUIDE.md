# Goedu-Split 1.0.1 사용설명서

Goedu-Split은 성취평가 결과를 빠르게 확인하고, 문항별 예상정답률과 NEIS 입력표 작성을 돕는 Windows용 프로그램입니다.

제작자: 이준서  
버전: 1.0.1  
배포 파일: `Goedu-Split-1.0.1.zip`

## 1. 프로그램 시작하기

1. `Goedu-Split-1.0.1.zip`의 압축을 풉니다.
2. 압축을 푼 폴더 안의 `Goedu-Split.exe`를 더블클릭합니다.
3. Windows 보안 안내가 나오면 **추가 정보 > 실행**을 선택합니다.

주의: `Goedu-Split.exe`만 따로 꺼내면 실행되지 않을 수 있습니다. 압축을 푼 폴더 전체를 그대로 둔 상태에서 실행해 주세요.

## 2. 전체 사용 흐름

Goedu-Split은 두 가지 방식으로 사용할 수 있습니다.

첫째, 정오표와 문항정보표를 넣어 성취평가 결과를 분석합니다.  
둘째, 자료를 모두 넣지 않아도 **예상정답률 입력** 탭에서 문항별 예상정답률과 NEIS 입력표를 작성합니다.

기본 분석 순서:

1. 왼쪽 **입력 데이터**에서 `학생답 정오표 data` 엑셀을 선택합니다.
2. `문항정보표` 엑셀을 선택합니다.
3. 필요하면 `예상추정분할점수 조회` 엑셀 또는 수행평가 결과를 추가합니다.
4. **분석 실행**을 누릅니다.
5. 오른쪽 탭에서 분석 결과를 확인합니다.

자주 쓰는 탭:

- **Data**: 불러온 원자료 확인
- **학생 포트폴리오**: 학생별 결과 확인
- **전체 성취도 분석**: 성취수준별 인원과 비율 확인
- **문항 분석**: 문항별 정답률, 변별도, 난이도 확인
- **성취수준별 답지반응 분포**: A~E 수준별 답지 선택 경향 확인
- **예상정답률 입력**: 문항별 예상정답률 작성, NEIS 입력표 생성
- **AI 문항 검토**: 로컬 초안 또는 Codex CLI를 활용한 문항 검토

## 3. 예상정답률 입력 탭의 역할

**예상정답률 입력** 탭은 NEIS에 넣을 문항별 예상정답률을 정리하는 공간입니다.

이 탭에서 할 수 있는 일:

- 문항번호와 배점 확인
- 선택형/서답형 유형 정리
- 난이도와 목표 성취수준 조정
- A, B, C, D, E 수준 학생의 예상 정답률 입력
- 전체 예상 분할점수 확인
- NEIS 입력표 만들기
- 시험지 반영표 만들기
- 문항 구성안 또는 근거 엑셀 저장

자료를 아직 모두 올리지 않았더라도 이 탭은 사용할 수 있습니다. 문항과 배점, 목표 성취수준을 직접 입력해 예상정답률표를 만들 수 있습니다.

## 4. 예상정답률 표 읽는 법

표의 주요 열은 다음과 같습니다.

- **선택**: 문항 삭제, 선택 문항 작업 등에 사용할 체크 표시입니다.
- **문항**: 문항 번호입니다.
- **배점**: 해당 문항의 점수입니다.
- **유형**: 선택형 또는 서답형입니다.
- **난이도**: 쉬움, 보통, 어려움 등을 선택합니다.
- **목표**: 이 문항이 주로 변별하려는 목표 성취수준입니다.
- **가상학생**: 예상정답률 계산에 쓰는 가상 학생 수입니다.
- **A~E**: 각 성취수준 학생이 이 문항을 맞힐 것으로 예상되는 비율입니다.

예를 들어 A열에 `3/3 100%`가 보이면, A 수준 가상학생 3명 중 3명이 맞힌다고 보는 뜻입니다. D열에 `1/3 33.3%`가 보이면, D 수준 가상학생 3명 중 1명이 맞힌다고 보는 뜻입니다.

## 5. 목표 성취수준과 예상정답률

각 문항의 **목표**를 A, B, C, D, E 중 하나로 바꾸면 A~E 예상정답률이 함께 바뀝니다.

예를 들어 목표를 **C**로 바꾸면, C 수준 학생이 어느 정도 맞힐 수 있고 D/E 수준 학생은 더 낮게 맞히는 형태로 예상정답률이 조정됩니다. 이 값은 절대적인 정답이 아니라, 교사가 문항 난이도와 성취기준을 보며 조정하는 출발점입니다.

사용 순서:

1. 문항의 배점이 맞는지 확인합니다.
2. 난이도를 확인합니다.
3. 목표 성취수준을 선택합니다.
4. A~E 예상정답률이 자연스러운지 봅니다.
5. 필요하면 A~E 셀을 직접 클릭하거나 조정해 수정합니다.
6. 상단의 **전체 예상 분할점수**가 의도한 흐름과 맞는지 확인합니다.

## 6. 상단 전체 예상 분할점수 이해하기

상단의 **전체 예상 분할점수**는 체크한 문항만 계산한 값이 아닙니다. 표에 있는 전체 문항을 기준으로 계산합니다.

쉽게 말하면 이렇습니다.

1. 각 문항의 배점과 A 수준 예상정답률을 곱합니다.
2. 모든 문항을 더하면 A/B 분할점수의 예상값이 됩니다.
3. 같은 방식으로 B, C, D 수준도 계산합니다.
4. 그래서 A/B, B/C, C/D, D/E 점수가 표시됩니다.

예를 들어 4점 문항에서 B 수준 예상정답률이 75%라면, 그 문항은 B/C 분할점수 계산에 3점 정도 기여합니다. 이런 식으로 모든 문항을 합산합니다.

따라서 화면에서 한 문항만 체크되어 있어도 상단 분할점수는 전체 문항 기준입니다. 체크는 선택 작업을 위한 것이고, 상단 분할점수 계산 범위를 줄이는 기능은 아닙니다.

## 7. NEIS 입력표 만들기

NEIS에 넣을 표를 만들 때는 **예상정답률 입력** 탭의 오른쪽 위 버튼을 사용합니다.

권장 작업 순서:

1. 문항번호와 배점을 확인합니다.
2. 각 문항의 유형, 난이도, 목표 성취수준을 확인합니다.
3. A~E 예상정답률을 검토하고 필요하면 수정합니다.
4. 상단 **전체 예상 분할점수**가 학교에서 의도한 A/B, B/C, C/D, D/E 흐름과 맞는지 확인합니다.
5. **NEIS 입력표** 버튼을 누릅니다.
6. 만들어진 표를 확인한 뒤 NEIS 입력 작업에 활용합니다.

NEIS 입력표를 볼 때 확인할 점:

- 문항 수가 실제 시험 문항 수와 같은가?
- 배점 합계가 실제 총점과 같은가?
- A~E 예상정답률이 지나치게 한쪽으로 치우치지 않았는가?
- 쉬운 문항인데 E 수준 정답률이 너무 낮거나, 어려운 문항인데 A 수준 정답률이 너무 낮지 않은가?
- 목표 성취수준이 문항의 성취기준과 맞는가?
- 상단 전체 예상 분할점수가 학교에서 예상한 분할점수와 크게 어긋나지 않는가?

## 8. NEIS표 활용 팁

NEIS 입력표는 그대로 복사하기 전에 한 번 더 검토하는 것이 좋습니다.

특히 다음 상황은 다시 봐 주세요.

- A/B는 높은데 B/C, C/D가 비정상적으로 낮은 경우
- 특정 문항 하나가 전체 분할점수를 지나치게 흔드는 경우
- 서답형 고배점 문항의 예상정답률이 너무 낙관적이거나 비관적인 경우
- 목표 성취수준을 바꿨는데 A~E 예상정답률을 아직 검토하지 않은 경우
- 문항정보표에서 배점이 잘못 들어온 경우

상단의 전체 예상 분할점수는 최종 결재값을 대신하는 것이 아니라, 현재 예상정답률표가 만들어내는 결과를 빠르게 보여주는 점검용 값입니다.

## 9. 시험지 반영표와 근거 엑셀

**시험지 반영** 버튼은 문항별 예상정답률 정보를 시험지 검토용으로 정리할 때 사용합니다.

**근거 엑셀**은 예상정답률을 어떻게 잡았는지 나중에 다시 확인하거나 협의 자료로 남길 때 유용합니다.

권장 활용:

- 교과협의회 전: 예상정답률 초안을 만들고 근거 엑셀 저장
- 협의 중: 목표 성취수준과 A~E 예상정답률 조정
- 협의 후: NEIS 입력표와 근거 엑셀을 함께 보관

## 10. 작업 저장과 불러오기

예상정답률을 조정하다가 나중에 이어서 작업하려면 **작업 저장**을 사용합니다.

1. 예상정답률을 조정합니다.
2. **작업 저장**을 누릅니다.
3. 다음에 다시 열 때 **작업 불러오기**를 누릅니다.

작업 저장 파일에는 문항별 목표, 난이도, 예상정답률 조정값이 포함됩니다. 같은 시험의 검토를 이어갈 때 유용합니다.

## 11. AI 문항 검토

**AI 문항 검토** 탭은 문항정보와 성취기준을 바탕으로 검토 초안을 만드는 기능입니다.

기본값은 외부 전송이 없는 로컬 초안입니다. Codex CLI 클라우드 AI를 선택한 경우에만 Codex CLI를 통해 문항 검토 요청이 전송됩니다.

Codex CLI 클라우드 AI는 API Key를 입력하지 않습니다. Windows 터미널에서 ChatGPT 계정으로 한 번 로그인한 뒤 사용합니다.

```powershell
winget install OpenJS.NodeJS.LTS
npm install -g @openai/codex
where codex
codex --version
codex login
codex login status
```

`codex login status`가 `Logged in using ChatGPT`로 나오면 Goedu-Split에서 **AI 문항 검토 > AI 설정 > Codex CLI 클라우드 (OAuth) > 연결 테스트**를 누릅니다.

## 12. 개인정보와 보안

- 기본 분석은 선생님 PC 안에서 실행됩니다.
- 배포 zip에는 `.git`, `.env`, `.venv`, 실제 학생자료, API Key, 토큰 파일이 포함되지 않도록 점검했습니다.
- Codex CLI 클라우드 AI를 사용할 때는 선생님이 선택한 검토 자료가 Codex CLI로 처리됩니다.
- 공유용 자료를 만들 때는 학생 이름, 반/번호, 개인정보가 필요 이상 포함되지 않았는지 확인해 주세요.

## 13. 자주 묻는 질문

**Q. 예상정답률 탭의 위쪽 점수가 선택한 문항과 달라 보여요.**  
A. 정상입니다. 위쪽 **전체 예상 분할점수**는 체크한 문항이 아니라 표 전체 문항 기준입니다.

**Q. NEIS 입력표 버튼을 눌렀는데 먼저 무엇을 봐야 하나요?**  
A. 문항 수, 배점 합계, A~E 예상정답률, 전체 예상 분할점수를 먼저 확인해 주세요.

**Q. 목표 성취수준을 바꾸면 자동값을 그대로 써도 되나요?**  
A. 자동값은 출발점입니다. 문항의 실제 난이도와 성취기준에 맞게 교사가 한 번 더 조정하는 것을 권장합니다.

**Q. Codex CLI 연결 테스트가 실패합니다.**  
A. 터미널에서 `codex login status`를 먼저 확인해 주세요. ChatGPT 로그인 상태가 아니면 Goedu-Split에서도 연결할 수 없습니다.

**Q. 새 버전은 어떻게 받나요?**  
A. GitHub Releases에서 최신 zip을 내려받아 새 폴더에 압축을 풀고 실행하면 됩니다.
